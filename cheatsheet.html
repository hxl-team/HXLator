HXLator Cheat Sheet

Explanations of all technical terms used in HXLator. In addition to the short explanations provided here, some terms are marked by a book icon. Clicking this will take you to a more elaborate explanation, e.g., on WikiPedia.

TermShort Explanation
Feature Geographical features are things we find in the real world, both natural (lakes, rivers, mountains, etc.) and artificial (cities, streets, buildings, etc.). In HXL, we stick to the terminology of the Open Geospatial Consortium that uses the term feature both for geographical features and for their representations, e.g. in HXL, in KML, or as a shape on a map.
Linked Data Linked Data is a method of publishing structured data based on standard Web technologies such as HTTP and URIs so that it can be interlinked and become richer and reusable. It adopts the idea of interlinked documents from the human-readable Web to raw data, interlinking datasets (usually represented in RDF) and making the queryable.
Mapping/translator We refer to a configuration to translate a spreadsheet to HXL as a mapping or translator. Once you have converted a spreadsheet and submitted your data for approval, the mapping is stored for future re-use, so that you do not have to go through the whole process again if you want to HXLate another spreadsheet that has the same structure or uses the same template.
URI/URL Short for Uniform Resource Identifier. URIs include URLs (Uniform Resource Locators) such as http://humanitarianresponse.info that you use whenever you visit a webpage. HXL builds on the Linked Data approach, which uses URLs as identifiers for things. For HXL, this means that we have a URL for each affected people's location (e.g., http://hxl.humanitarianresponse.info/data/locations/apl/bfa/UNHCR-POC-80), admin unit (e.g., http://hxl.humanitarianresponse.info/data/locations/admin/bfa/BFA056001), person (e.g., http://hxl.humanitarianresponse.info/data/persons/unocha/cj_hendrix ), organisation (e.g., http://hxl.humanitarianresponse.info/data/organisations/unocha), and so on. As you can see, URLs are perfect for publishing data on the web, where they both uniquely identify a thing and tell us where to find information about this thing.
RDF The Resource Description Framework (RDF) is a data model defined by W3C specifications that allows to make statements (in the form of triples) about resources. Unlike other technologies like XML, RDF is an abstract model that can be encoded in various file formats such as N-Triples, Turtle, or RDF/XML.
SPARQL The SPARQL Protocol and RDF Query Language enables queries over RDF data, comparable (and not too different in syntax) to SQL for relational databases.
SPARQL endpoint A SPARQL endpoint is a HTTP address published by a triple store that clients can send SPARQL requests to, which will then be executed by the store. The results are then returned by HTTP again and can be encoded in a variety of formats such as XML, CSV, or JSON. The client can specifiy the expected result format in its query header.
Triple In the context of RDF and Linked Data, triples are statements consisting of subject, predicate, and object, such as Batman knows Robin. The triples are made machine-readable by replacing the parts of the statment with URIs which contain more information (e.g. about Batman, Robin, or the knows predicate).
Triple store A triple store is a database optimized for storage, indexing and querying triples. It usually offers a SPARQL endpoint to query (and possibly insert, update, and delete) data.
Turtle Turtle is an easy-to-read and compact syntax for RDF.
